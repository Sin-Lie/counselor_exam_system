## 1. 整体架构

```
                        用户浏览器
                            |
                         🖥️  CentOS 7.9 服务器（完全离线）
                            |
                       [Nginx 容器]  ← 端口 80（HTTP）
                       /             \
                 提供前端静态文件    反向代理 /api 和 /media
                       |              |
                  [Vue 构建产物]   [Django 后端容器] ← 端口 8000
                                     /            \
                               [MySQL 容器]    [Redis 容器]
                               端口 3306       端口 6379
```

**四个容器：**

| 容器        | 用途               | 基础镜像             | 暴露端口        |
| --------- | ---------------- | ---------------- | ----------- |
| `nginx`   | 提供前端页面 + 反向代理到后端 | nginx:alpine     | 80          |
| `backend` | Django API 服务    | python:3.11-slim | 8000（内部）    |
| `mysql`   | 关系数据库            | mysql:8.0        | 3307（映射到主机） |
| `redis`   | 缓存/考试计时/防作弊      | redis:7-alpine   | 6379        |

**数据持久化（Docker 命名卷）：**

- `counselor_mysql_data` → MySQL 数据文件
- `counselor_redis_data` → Redis 数据文件
- `counselor_media` → 上传的媒体文件（学生照片等）
- `counselor_logs` → Django 运行日志
- `counselor_static` → Django 静态文件

## 2. 环境要求

### 2.1 离线服务器（CentOS 7.9）

- **操作系统:** CentOS 7.9（内核 3.10 以上）
- **CPU:** 至少 2 核
- **内存:** 至少 4GB（推荐 8GB）
- **磁盘:** 至少 20GB 可用空间
- **网络:** 完全离线（无互联网访问）

查看系统版本：

```bash
cat /etc/centos-release
uname -r
```

### 2.2 联网机器 —— 你的 Windows 电脑（用于制作物料包）

就用你现在用的这台 Windows 电脑，在上面：

- 安装 Docker Desktop（用来拉取和构建镜像）
- 下载 Docker 离线安装包（RPM 文件）
- 拉取 Docker 基础镜像
- 构建项目 Docker 镜像并导出

> **你的 Windows 电脑需要提前安装：**
> 
> - **Docker Desktop**（https://www.docker.com/products/docker-desktop/）
>   安装后启动，确保右下角 Docker 图标运行中
> - **PowerShell**（Win 10/11 自带）

## 3. 准备工作：在联网机器上制作离线物料包

> **核心思路：** 所有需要互联网的操作，全部在联网机器上提前准备好，打包成一个"物料包"，复制到离线服务器上直接使用。

### 3.1 下载 Docker 离线安装包（RPM 方式）

在 Windows 上打开浏览器，手动下载以下 RPM 文件（el7 就是 CentOS 7 的版本）。

**下载地址表（点击链接下载）：**

| 包名        | 下载链接                                                               |
| --------- | ------------------------------------------------------------------ |
| docker-ce | https://download.docker.com/linux/centos/7/x86_64/stable/Packages/ |

打开上面的链接，找到最新版本的以下 5 个 RPM 包下载：

| 需要下载的文件                                    | 说明               |
| ------------------------------------------ | ---------------- |
| `docker-ce-版本号.el7.x86_64.rpm`             | Docker 引擎主程序     |
| `docker-ce-cli-版本号.el7.x86_64.rpm`         | Docker CLI 命令行工具 |
| `containerd.io-版本号.el7.x86_64.rpm`         | 容器运行时            |
| `docker-buildx-plugin-版本号.el7.x86_64.rpm`  | 构建插件             |
| `docker-compose-plugin-版本号.el7.x86_64.rpm` | Compose 插件       |

> **找文件技巧：** 页面列表按文件名排序，找名字里带 `el7` 的最新版本。例如：
> 
> - `docker-ce-24.0.9-1.el7.x86_64.rpm`
> - `docker-ce-cli-24.0.9-1.el7.x86_64.rpm`
> - `containerd.io-1.6.28-3.1.el7.x86_64.rpm`
> - `docker-buildx-plugin-0.11.2-1.el7.x86_64.rpm`
> - `docker-compose-plugin-2.24.0-1.el7.x86_64.rpm`

如果上面那个页面找不到 `containerd.io`，去这里下载：
https://download.docker.com/linux/centos/7/x86_64/stable/Packages/

还需要下载一个依赖包 **container-selinux**：

- 下载地址：http://mirror.centos.org/centos/7/extras/x86_64/Packages/
- 找文件名如 `container-selinux-2.119.2-1.911c772.el7_8.noarch.rpm`

**下载完成后，把这 6 个 .rpm 文件都放到同一个文件夹里，例如：**

```
D:\离线部署\docker-rpms\
├── docker-ce-24.0.9-1.el7.x86_64.rpm
├── docker-ce-cli-24.0.9-1.el7.x86_64.rpm
├── containerd.io-1.6.28-3.1.el7.x86_64.rpm
├── docker-buildx-plugin-0.11.2-1.el7.x86_64.rpm
├── docker-compose-plugin-2.24.0-1.el7.x86_64.rpm
└── container-selinux-2.119.2-1.911c772.el7_8.noarch.rpm
```

### 3.2 下载 Docker Compose 离线二进制

在 Windows PowerShell 中下载：

```powershell
# 创建存放目录
mkdir D:\离线部署\docker-compose-pack

# 下载 Docker Compose 二进制
Invoke-WebRequest -Uri "https://github.com/docker/compose/releases/latest/download/docker-compose-linux-x86_64" -OutFile "D:\离线部署\docker-compose-pack\docker-compose"
```

> 如果下载慢，可以用国内镜像：
> 
> ```powershell
> Invoke-WebRequest -Uri "https://get.daocloud.io/docker/compose/releases/download/v2.24.0/docker-compose-linux-x86_64" -OutFile "D:\离线部署\docker-compose-pack\docker-compose"
> ```

下载完成后得到一个 `docker-compose` 文件（没有后缀名），这就是 Linux 上的可执行文件。

### 3.3 拉取并导出 Docker 基础镜像

> **前提：** 确保你的 Windows 电脑已安装并启动了 Docker Desktop（任务栏右下角有鲸鱼图标）。

打开 PowerShell，执行以下命令：

```powershell
# 拉取所有需要的基础镜像
docker pull mysql:8.0
docker pull redis:7-alpine
docker pull python:3.11-slim
docker pull node:20-alpine
docker pull nginx:alpine

# 查看是否全部拉取成功
docker images
```

> **如果拉取很慢：** 可以配置 Docker Desktop 的镜像加速器。
> 打开 Docker Desktop → 设置 → Docker Engine → 添加以下配置：
> 
> ```json
> {
>   "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]
> }
> ```
> 
> 点击 "Apply & Restart" 后再重新拉取。

拉取完成后，将所有基础镜像导出为一个 tar 文件：

```powershell
# 导出基础镜像（在 D 盘创建离线目录）
mkdir D:\离线部署\ -Force
docker save mysql:8.0 redis:7-alpine python:3.11-slim node:20-alpine nginx:alpine -o D:\离线部署\base-images.tar

# 查看大小
ls D:\离线部署\base-images.tar
```

### 3.4 构建项目镜像并导出

在你的 Windows 电脑上，直接在项目目录中构建：

```powershell
# 进入项目目录（根据你的实际路径修改）
cd C:\Users\Sin\Desktop\生情熟知V1

# 构建后端镜像
docker build -t counselor-backend:latest .\backend

# 构建前端镜像
docker build -t counselor-frontend:latest .\frontend

# 查看构建好的镜像
docker images | findstr counselor
```

> **如果 build 报错 "Cannot connect to Docker"：**
> 确保 Docker Desktop 已启动（任务栏右下角有鲸鱼图标）。

构建成功后，导出项目镜像：

```powershell
# 导出到之前创建的离线目录
docker save counselor-backend:latest counselor-frontend:latest -o D:\离线部署\project-images.tar

# 查看大小
ls D:\离线部署\project-images.tar
```

### 3.5 最终物料清单

回到 `D:\离线部署\` 目录，确认以下所有文件都已齐备：

```
D:\离线部署\
├── docker-rpms\                    # 文件夹，里面放 6 个 .rpm 文件
│   ├── docker-ce-24.0.9-1.el7.x86_64.rpm
│   ├── docker-ce-cli-24.0.9-1.el7.x86_64.rpm
│   ├── containerd.io-1.6.28-3.1.el7.x86_64.rpm
│   ├── docker-buildx-plugin-0.11.2-1.el7.x86_64.rpm
│   ├── docker-compose-plugin-2.24.0-1.el7.x86_64.rpm
│   └── container-selinux-2.119.2-1.911c772.el7_8.noarch.rpm
│
├── docker-compose-pack\            # 文件夹
│   └── docker-compose              # 单个文件（没有后缀名）
│
├── base-images.tar                 # 基础镜像（约 1.5GB）
└── project-images.tar              # 项目镜像（约 1GB）
```

**加上项目文件本身**（`生情熟知V1` 文件夹），这就是需要复制到离线服务器的全部内容。

> **传输方式：**
> 
> - 用 **U 盘** 把所有文件拷贝到离线服务器
> - 或在局域网内用 scp 传输
> - 总大小约 3-5GB，建议用 USB 3.0 的 U 盘或移动硬盘

## 5. 配置修改说明

### 5.1 修改密码和密钥

部署前，**必须**修改 `.env.prod` 文件中的以下值：

| 变量                    | 说明            | 修改建议                |
| --------------------- | ------------- | ------------------- |
| `DJANGO_SECRET_KEY`   | Django 安全密钥   | 生成一个随机字符串，至少 50 个字符 |
| `MYSQL_ROOT_PASSWORD` | MySQL root 密码 | 设置强密码               |
| `MYSQL_PASSWORD`      | 业务数据库用户密码     | 设置强密码               |

生成随机密钥的方法（在联网机器或服务器上执行）：

```bash
# 生成 Django SECRET_KEY
python3 -c "import secrets; print(''.join(secrets.choice('abcdefghijklmnopqrstuvwxyz0123456789!@#$%^&*(-_=+)') for i in range(50)))"

# 生成数据库密码（16位随机密码）
openssl rand -base64 12
```

### 5.2 修改域名/IP 配置

如果服务器有域名，修改 `.env.prod` 中的：

```
DJANGO_ALLOWED_HOSTS=你的域名或服务器IP
```

例如：`DJANGO_ALLOWED_HOSTS=192.168.1.100`

> 如果没有域名用 IP 地址即可，多个地址用逗号分隔。

### 5.3 SSL/HTTPS 配置（初次部署可跳过）

项目默认关闭 HTTPS。如果后续配置了域名和 SSL 证书：

1. 将证书文件放到服务器上

2. 修改 `frontend/nginx.conf` 添加 SSL 配置

3. 修改 `.env.prod` 开启：
   
   ```
   DJANGO_SECURE_SSL_REDIRECT=True
   DJANGO_SESSION_COOKIE_SECURE=True
   DJANGO_CSRF_COOKIE_SECURE=True
   ```

## 6. 将物料包上传到离线服务器

**如果服务器在局域网内（用 scp 传输）：**

在 Windows PowerShell 中执行：

```powershell
# 假设你的 CentOS 服务器 IP 为 192.168.1.100
# 上传 RPM 文件夹
scp -r D:\离线部署\docker-rpms root@192.168.1.100:/opt/

# 上传 Docker Compose 文件夹
scp -r D:\离线部署\docker-compose-pack root@192.168.1.100:/opt/

# 上传镜像文件（较大，需要几分钟）
scp D:\离线部署\base-images.tar root@192.168.1.100:/opt/
scp D:\离线部署\project-images.tar root@192.168.1.100:/opt/

# 上传项目文件
scp -r "C:\Users\Sin\Desktop\生情熟知V1" root@192.168.1.100:/opt/
```

> 如果 `scp` 不能用，检查 Windows 是否安装了 OpenSSH 客户端：
> 
> ```powershell
> Get-WindowsCapability -Online | Where-Object Name -like 'OpenSSH.Client*'
> ```

**如果用 U 盘：**

```bash
# 在服务器上，插入 U 盘后挂载
sudo mkdir -p /mnt/usb
sudo mount /dev/sdb1 /mnt/usb

# 复制所有文件
sudo cp -r /mnt/usb/docker-rpms /opt/
sudo cp -r /mnt/usb/docker-compose-pack /opt/
sudo cp /mnt/usb/base-images.tar /opt/
sudo cp /mnt/usb/project-images.tar /opt/
sudo cp -r /mnt/usb/生情熟知V1 /opt/

# 卸载 U 盘
sudo umount /mnt/usb
```

> 如果上面的 `/dev/sdb1` 不对，先运行 `lsblk` 查看 U 盘的设备名。

## 7. 在离线服务器上部署

### 第1步：离线安装 Docker（使用 RPM 包）

```bash
cd /opt

# 进入 RPM 包目录（文件通过 U 盘或 scp 提前复制到这里）
cd /opt/docker-rpms/

# 离线安装所有 RPM 包（自动解决依赖）
sudo yum localinstall -y *.rpm

# 启动 Docker 并设置为开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
```

### 第2步：安装 Docker Compose

```bash
# 创建 Docker CLI 插件目录
sudo mkdir -p /usr/local/lib/docker/cli-plugins/

# 复制 docker-compose 二进制到插件目录
sudo cp /opt/docker-compose-pack/docker-compose /usr/local/lib/docker/cli-plugins/docker-compose

# 添加执行权限
sudo chmod +x /usr/local/lib/docker/cli-plugins/docker-compose

# 验证
docker compose version
```

### 第3步：导入 Docker 镜像

```bash
# 导入基础镜像（约 1.5GB，需要几分钟）
docker load -i base-images.tar

# 导入项目镜像
docker load -i project-images.tar

# 验证所有镜像已导入
docker images
```

应该能看到以下镜像：

```
REPOSITORY           TAG       IMAGE ID       SIZE
counselor-backend    latest    ...            ... MB
counselor-frontend   latest    ...            ... MB
mysql                8.0       ...            600 MB
redis                7-alpine  ...            30 MB
python               3.11-slim ...            200 MB
node                 20-alpine ...            130 MB
nginx                alpine    ...            40 MB
```

> 如果缺少某个镜像，可以在联网机器上重新拉取导出，或者检查是否漏了步骤。

### 第4步：配置环境变量

```bash
cd /opt/生情熟知V1

# 确保 .env.prod 已正确配置
vim .env.prod

# 【重要】修改密码和密钥（参照第5节）

# 设置文件权限，防止其他用户读取密码
chmod 600 .env.prod
```

### 第5步：启动所有容器

```bash
cd /opt/生情熟知V1

# 启动所有容器（--no-build 表示跳过构建，直接使用已导入的镜像）
docker compose --env-file .env.prod up -d --no-build
```

### 第6步：查看启动日志

```bash
# 查看所有容器的启动日志
docker compose logs -f

# 只查看后端的日志
docker compose logs -f backend

# 只查看 MySQL 的日志
docker compose logs -f mysql
```

看到以下关键日志行说明启动成功：

**MySQL:**

```
[System] [MY-010931] [Server] /usr/sbin/mysqld: ready for connections.
```

**Backend:**

```
MySQL 数据库就绪！
执行数据库迁移...
收集静态文件...
启动 Gunicorn 服务器...
[INFO] Listening at: http://0.0.0.0:8000
```

**Nginx:**

```
/docker-entrypoint.sh: Configuration complete; ready for start up
```

### 第7步：验证部署

在浏览器中访问服务器 IP：

```
http://你的服务器IP
```

应该能看到辅导员考试系统的登录页面。

> **打不开？** 检查防火墙：
> 
> ```bash
> # 放行 80 端口
> sudo firewall-cmd --permanent --add-port=80/tcp
> sudo firewall-cmd --reload
> ```

### 第8步：创建管理员用户（首次部署必做）

```bash
# 在后端容器中创建超级管理员
docker compose exec backend python manage.py createsuperuser
```

按提示输入用户名、邮箱、密码。

## 8. 日常管理命令

### 8.1 查看容器状态

```bash
# 查看所有容器运行状态
docker compose ps

# 查看容器资源占用
docker stats
```

### 8.2 启动 / 停止 / 重启

```bash
# 启动所有容器
docker compose --env-file .env.prod start

# 停止所有容器（数据不会丢失）
docker compose stop

# 重启所有容器
docker compose restart

# 完全停止并删除容器（数据保留在卷中）
docker compose down
```

### 8.3 查看日志

```bash
# 实时查看所有日志
docker compose logs -f

# 查看后端最后 100 行日志
docker compose logs --tail=100 backend

# 查看 Nginx 访问日志
docker compose logs -f nginx
```

### 8.4 进入容器内部

```bash
# 进入后端容器
docker compose exec backend bash

# 进入 MySQL 容器
docker compose exec mysql mysql -u root -p

# 进入 Redis 容器
docker compose exec redis redis-cli
```

### 8.5 更新项目（代码修改后重新部署）

修改代码后，需要在 Windows 上重新构建镜像并导出：


### 9.1 备份数据库

```bash
# 创建备份目录
mkdir -p /opt/backup

# 查看 MySQL 密码（从 .env.prod 中获取）
MYSQL_PWD=$(grep MYSQL_ROOT_PASSWORD /opt/生情熟知V1/.env.prod | cut -d= -f2)

# 备份 MySQL 数据库（排出密码到文件避免特殊字符问题）
docker compose exec mysql mysqldump -u root -p"${MYSQL_ROOT_PASSWORD}" counselor_exam_system > /opt/backup/db_$(date +%Y%m%d_%H%M%S).sql

# 保留最近 30 天的备份，删除更早的
find /opt/backup -name "db_*.sql" -mtime +30 -delete
```

```bash
# 编辑定时任务
crontab -e

```

## 10. 常见问题排查

### ❓ 问题1：docker compose 命令报错

**现象：** `command not found: docker compose`

**解决：** 检查 Docker Compose 是否正确安装：

```bash
ls -la /usr/local/lib/docker/cli-plugins/docker-compose
# 如果没有，重新执行第 7 章第 2 步
```

### ❓ 问题2：端口被占用

**现象：** `Error: starting userland proxy: listen tcp4 0.0.0.0:80: bind: address already in use`

**解决：** 服务器上已有程序占用了 80 端口：

```bash
# 查看什么程序占用了端口
sudo netstat -tlnp | grep :80

# 如果是 Apache/httpd，可以先停掉
sudo systemctl stop httpd
sudo systemctl disable httpd
```

### ❓ 问题3：MySQL 连接失败

**现象：** 后端日志显示 `Can't connect to MySQL server`

**解决：** MySQL 首次启动需要初始化数据库，耗时较长：

```bash
# 查看 MySQL 日志
docker compose logs mysql

# 等 1-2 分钟后重启后端
docker compose restart backend
```

### ❓ 问题4：页面空白或 404

**现象：** 能访问页面但空白，或刷新页面时 404

**解决：** 检查 Nginx 是否正确配置 SPA 路由回退：

```bash
docker compose logs nginx
```

确认 `nginx.conf` 中有 `try_files $uri $uri/ /index.html;`（我们已配置好）。

### ❓ 问题5：上传文件过大

**现象：** 上传 Excel 文件时失败

**解决：** 修改 `frontend/nginx.conf` 中的 `client_max_body_size` 值，然后在联网机器上重新构建前端镜像并导入服务器。

### ❓ 问题6：防火墙导致无法访问

**现象：** 容器正常运行但浏览器无法访问

**解决：** CentOS 7 默认开启防火墙，需要放行端口：

```bash
# 查看防火墙状态
sudo firewall-cmd --state

# 放行 80 端口
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload

# 查看已放行的端口
sudo firewall-cmd --list-all
```

### ❓ 问题7：docker load 导入镜像很慢

**现象：** `docker load -i base-images.tar` 执行时间很长

**解决：** 属于正常现象，基础镜像约 1.5GB，导入需要 3-5 分钟。如果磁盘 I/O 慢可能更久，耐心等待。

### ❓ 问题8：缺少某个 Docker 镜像

**现象：** `docker images` 中缺少某个镜像，启动时报错

**解决：** 在联网机器上单独拉取缺少的镜像并导出：

```bash
# 在联网机器上
docker pull 缺少的镜像名:标签
docker save 缺少的镜像名:标签 -o missing.tar

# 传到离线服务器后
docker load -i missing.tar
```
